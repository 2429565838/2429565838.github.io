如何进行环境区分

发表于2024-05-08|更新于2024-05-08
|字数总计:347|阅读时长:1分钟|阅读量:

这个就是环境文件:在根目录下

.env.development

.env.production

.env.staging

  1. NODE_ENV = production：这行配置设置了 Node.js 进程的环境变量为 production。在 Vue.js 应用中，这通常用于指示应用程序当前处于生产环境，以便在构建时进行相关的优化和调整。
  2. ENV = 'staging'：这行配置设置了一个自定义的环境变量 ENV 的值为 'staging'，表示当前环境为测试或者预发布环境。这种自定义的环境变量通常用于标识当前应用程序所处的特定环境，以便在应用程序中进行相应的调整和配置。
  3. VUE_APP_BASE_API = '/stage-api'：这行配置设置了一个名为 VUE_APP_BASE_API 的环境变量的值为 '/stage-api'，用于指定应用程序的基础 API 地址。在 Vue.js 应用中，以 VUE_APP_ 开头的环境变量会被 Vue CLI 自动注入到应用程序中，并可以在应用程序中通过 process.env.VUE_APP_BASE_API 来获取。在这里，VUE_APP_BASE_API 可能用于指定应用程序在测试环境中要调用的后端 API 的地址。
